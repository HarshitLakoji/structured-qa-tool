Question: What is a Data Structure? Explain the difference between primitive and non-primitive data structures with examples.
Answer: A Data Structure is a way of organizing and storing data in a computer program so that it can be accessed and used efficiently. It provides a means of managing large amounts of data, enabling efficient searching, sorting, insertion and deletion of data.

Data structures can be categorized into two types: Primitive data structures and non-primitive data structures.

**Primitive Data Structures:**
Primitive data structures are the most basic data structures available in a programming language, such as integers, floating-point numbers, characters, and Booleans. These data structures are used to store a single value.

Example:
```c
int num = 10;          // Integer
float pi = 3.14;       // Float
char letter = 'A';     // Character
bool isHappy = true;   // Boolean
```
**Non-Primitive Data Structures:**
Non-primitive data structures are more complex and are built using primitive data types. They are further divided into various categories, such as Linear Data Structures (arrays, linked lists, stacks, queues) and Non-Linear Data Structures (trees, graphs, hash tables).

Example:
```c
// Array (Linear Data Structure)
int arr[5] = {1, 2, 3, 4, 5};

// Stack (Linear Data Structure)

s.push(1);
s.push(2);
s.push(3);
```

In summary, primitive data structures are basic and store a single value, while non-primitive data structures are complex and built using primitive data types to store multiple values.

Question: Explain the Tower of Hanoi problem and list its rules.
Answer: The Tower of Hanoi problem is a mathematical puzzle where we have three rods (A, B, and C) and N disks. Initially, all the disks are stacked in decreasing value of diameter, i.e., the smallest disk is placed on the top and they are on rod A. The objective of the puzzle is to move the entire stack to another rod (here considered C), obeying the following simple rules:

1. Only one disk can be moved among the towers at any given time.
2. Only the "top" disk can be removed.
3. No large disk can sit over a small disk.

Question: Explain memory representation of a one-dimensional array.
Answer: Based on the provided context, the memory representation of a one-dimensional array is as follows:

An array requires memory blocks to store its elements. Since the array is one-dimensional, its elements are stored in contiguous memory blocks. 

For example, consider a one-dimensional array 'num' with 10 elements of type 'int'. Each element takes 2 bytes of memory. If the starting address of the first memory block is 2000, then the next elements will get the next contiguous 4 bytes memory block (2 bytes per element). 

The memory address of each element will be the starting address of the first memory block plus the number of bytes occupied by the previous elements. In this case, the memory addresses of the elements will be:

- First element: 2000
- Second element: 2002
- Third element: 2004
- ...
- Tenth element: 2018

This is the memory representation of a one-dimensional array.

Question: Derive and explain the formula to calculate the address of A[i] in a 1D array.
Answer: The formula to calculate the address of A[i] in a 1D array is:

Address of A[i] = Base_Address + W(i - Lower_Bound)

Where:
- Base_Address is the address of the first element in the array.
- W is the weight (size) of a data type.
- i is the index of the array element to be calculated.
- Lower_Bound is the index of the first element.

This formula is derived from the fact that each element in the array occupies a contiguous memory block, and the address of each element is the base address plus the size of the data type multiplied by the difference between the index and the lower bound.

Question: Explain the Bubble Sort algorithm with its time complexity. Why is it not suitable for large datasets?
Answer: The Bubble Sort algorithm is a simple sorting algorithm that works by repeatedly iterating through a list of elements, comparing each pair of adjacent elements and swapping them if they are in the wrong order. This process continues until no more swaps are needed, indicating that the list is sorted.

The time complexity of the Bubble Sort algorithm is O(n^2), where n is the number of elements in the list. This is because in the worst-case scenario, the algorithm has to compare each pair of elements, resulting in n*(n-1)/2 comparisons.

Bubble Sort is not suitable for large datasets because its average and worst-case complexity are O(n^2), which can be very slow for large lists. This is because the algorithm has to compare each pair of elements, resulting in a quadratic number of comparisons. As a result, the algorithm's performance degrades rapidly as the size of the input list increases.
